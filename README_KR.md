# GPU 가속 기반 멀티 모델 Callable Swap 통합 평가 엔진
> **언어(Language)**: [English](./README.md) | [한국어]
### 한-미 시장 전략적 리스크 분석: Hull-White vs. LMM 비교

본 시스템은 **NVIDIA GPU (CUDA)** 아키텍처를 활용하여 한국(KRW)과 미국(USD) 시장의 **Bermudan Callable Swap** 가치를 실시간으로 산출하고 비교 분석하는 전문 퀀트 엔진입니다.

## 프로젝트 구조
```text
.
├── main.py                 # 통합 실행부: 한-미 시장 데이터 연동 및 비교 리포트 생성
├── Datahandler.py          # 데이터 파이프라인: 한국은행(ECOS) 및 yfinance API 연동
├── Model_selection.py      # 엔진 연결부: QuantLib 기반 수익률 곡선(Yield Curve) 구축
├── HW_GPU.py               # CUDA 커널: Hull-White 단기금리 경로 생성 (PyCUDA)
├── LMM_GPU.py              # 병렬 시뮬레이터: Cholesky 상관관계 반영 LMM 생성 (CuPy)
├── LSM_pricer.py           # GPU LSM (HW): 단기금리 모델용 최적 정지 알고리즘
└── LSM_pricer_LMM.py       # GPU LSM (LMM): 다차원 선도금리 시장 모델용 LSM
```

## 한-미 시장 비교 분석 결과

본 엔진은 저금리 기조의 한국과 고금리 기조의 미국 시장 간의 리스크 불균형을 데이터로 입증합니다.

### [통합 리스크 메트릭스 리포트]


| 시장 | 모델 | 평가가치 (%) | 델타 (1bp) | 감마 (곡률) | 베가 (1%) | 헤지비율 |
| :--- | :---: | :---: | :---: | :---: | :---: | :---: |
| **KOREA** | **HW** | 2.2712% | 0.0186 | 0.0622 | 1.7767 | 0.4144 |
| (KRW) | **LMM** | 0.6221% | 0.0221 | 0.1531 | 0.0164 | 0.4917 |
| **USA** | **HW** | 2.1705% | 0.0184 | -0.0032 | 1.7891 | 0.4093 |
| (USD) | **LMM** | 1.0089% | 0.0232 | -0.9432 | 0.0159 | 0.5149 |

### 한-미 시장 교차 분석 및 인사이트

1. **모델 리스크의 수치화 (HW-LMM 스프레드)**:
   - 한국 시장에서 두 모델 간 **164.9bp라는 거대한 스프레드**가 관찰되었습니다.
   - **분석**: 단일 인자 모델인 Hull-White가 수익률 곡선의 구조적 뒤틀림(Twist)을 반영하지 못해 옵션 가치를 과대평가하는 경향을 보여주며, LMM이 이를 훨씬 정교하게 보정하고 있음을 입증합니다.

2. **미국 시장의 네거티브 컨벡시티(Negative Convexity) 실증**:
   - 미국 시장의 감마는 두 모델 모두에서 **음수(Negative)**를 기록했습니다 (HW: -0.0032, LMM: -0.9432).
   - **분석**: 미국 금리가 행사가(3.5%)에 근접함에 따라 조기행사 확률이 포화 상태에 도달하여 금리 상승 시 가치 상승폭이 둔화되는 현상이 포착되었습니다. 특히 LMM 모델에서 이러한 '감마 플립' 현상이 더욱 뚜렷하게 나타나 정교한 리스크 관리가 필수적임을 시사합니다.

3. **델타 민감도 및 헤지 효율성**:
   - 전반적으로 **LMM 모델이 HW보다 높은 델타와 헤지 비율(약 0.49~0.51)**을 제시합니다.
   - **분석**: 포트폴리오의 델타 중립을 위해서는 원금의 약 50% 수준의 일반 스왑 물량이 필요함을 의미합니다. 다차원 금리 변화를 반영하는 LMM의 특성상 리스크에 더욱 기민하게 반응하고 있습니다.

4. **GPU 연산 성능의 우위**:
   - RTX 3070 기준, 총 **8가지 대규모 시나리오** 시뮬레이션을 **26초 이내**에 완수했습니다.
   - **의미**: 기존 CPU 기반 시스템으로는 수 분이 소요될 연산을 실시간(Real-time) 수준으로 단축하여, 장중 시장 변동에 따른 즉각적인 리스크 재조정(Rebalancing)이 가능함을 보여줍니다.

## 성능 및 하드웨어 벤치마크

본 엔진은 고성능 소비자용 하드웨어에 최적화되어 있으며, 대규모 몬테카를로 시뮬레이션에서 실무급 처리 속도를 보여줍니다.

### [하드웨어 사양]
- **CPU**: AMD Ryzen 9 5950X (16코어, 3.4 GHz)
- **GPU**: NVIDIA GeForce RTX 3070 (8GB VRAM, Ampere 아키텍처)
- **운영체제**: Windows 10 / CUDA 13.x 기반

### [실행 성능 리포트]
- **시뮬레이션 규모**: 시나리오당 100,000개 경로 생성.
- **연산 범위**: 2개 국가 시장(KRW, USD)에 대해 2개 모델(HW, LMM)의 총 8가지 시나리오 동시 분석.
- **전체 실행 시간**: **약 25.20초**
  - 포함 내역: 실시간 데이터 수집 + 수익률 곡선 구축 + GPU 시뮬레이션 + LSM 후진 귀납법 + Greeks 산출.
- **실시간 대응력**: 데이터 수집 이후, 단일 모델에 대한 가치 평가 및 Greeks 산출은 **1초 미만**의 속도로 완결됩니다.

## 기술적 최적화 포인트
- **Zero-Copy VRAM 설계**: 경로 생성부터 LSM 회귀 분석까지 데이터를 CPU로 복사하지 않아 처리 속도를 극대화했습니다.
- **명시적 메모리 관리**: `free_all_blocks()`와 `gc.collect()`를 활용하여 반복적인 Greeks 산출 시 발생하는 `cudaErrorInvalidValue`를 완벽히 차단했습니다.
- **상관관계 엔지니어링**: GPU 기반 Cholesky 분해를 통해 실제 시장의 테너 간 상관관계를 반영한 LMM 경로를 생성합니다.

## 모델의 한계점 및 향후 과제

본 엔진은 기술적으로 높은 성능을 보이나, 데이터 및 모델링 측면에서 다음과 같은 한계점을 가집니다.

1. **변동성 곡면(Volatility Surface) 미반영**:
   - **현황**: 현재 엔진은 모든 테너와 행사가에 대해 동일한 변동성($\sigma$)을 사용하는 Flat Volatility 가정을 따릅니다.
   - **한계**: 실제 시장의 **Volatility Smile 및 Skew(왜도)** 현상을 반영하지 못하므로, Deep ITM 또는 Deep OTM 구간에서 옵션 가치가 실제 시장가와 괴리될 수 있습니다.
   - **개선 방향**: 향후 Swaption Volatility Cube 데이터를 연동하여 **SABR 모델 또는 Local Volatility 모델**로의 확장이 필요합니다.

2. **결정론적 파라미터**:
   - Hull-White의 평균회귀 속도($a$)와 LMM의 상관계수가 고정값으로 설정되어 있습니다. 실무 적용을 위해서는 시장 변동성 구조에 맞춘 실시간 **Calibration(최적화)** 로직이 추가되어야 합니다.

3. **데이버시스 리스크(Basis Risk)**:
   - 현재 단일 커브(Single-curve) 체계에서 평가가 이루어집니다. 현대 금융 관행인 **Multi-curve Bootstrapping**(조달 금리와 할인 금리의 분리)은 아직 구현되지 않았습니다.

4. **데이터 실시간성**:
   - ECOS 및 yfinance를 통한 일별 종가 데이터를 사용합니다. 초단위 리스크 관리가 필요한 트레이딩 데스크 적용을 위해서는 블룸버그/로이터 등 전문 터미널과의 실시간 API 연동이 필요합니다.
